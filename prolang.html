Chapter 6: Programming Languages 

Machine language An encoding system consisting of sequences of instructions that are encoded as numerical digits.
Debugging The process of locating and correcting errors that occur in code.
Assembler A program that is used to convert mnemonic expressions into machine language. Converting human-readable text into a format the computer can process.
Assembly language A mnemonic system for representing programs.
Translator A program used to translate other programs expressed in high-level primitives into machine-language programs.
Compiler A type of translator that compiles several machine instructions into short sequences to simulate the activity requested by a single high-level primitive. This program will "read" through all code before running the program.
Interpreter Programs similar to translators except they execute the instructions as they were translated instead of recording the translation for future use. These programs run through code as is written and will execute until it finds the first error.
Cross-platform software Software that is independent of the type of operating system being used. This software is compatible with different types of OS and as a result is able to be used more widely throughout networks with varying OS.
Natural language Languages such as: English, German, Spanish, etc.
Formal language Programming languages such as: Python, C++, C#, etc.
Machine independence Programming languages (primitives) that do not rely on the characteristics of any one particular machine.
High-level programming language Include many kinds of abstractions, ranging from the simple - constants, literals, and variables - to moderate complexity - statements, expressions, and control structures - to the esoteric - procedural units, modules, and libraries.
Object-oriented programming (OOP) A programming process where a software system is viewed as a collection fo units, called objects, each of which is capable of performing the actions that are immediately related to itself as well as requesting actions of other objects. Together, these objects interact to solve the problem at hand.
Scripting language Languages that are typically used to perform administrative tasks rather than to develop complex programs. The expression of such a task is known as a script. Example: A user of a PC might write a script to direct the execution of a sequence of programs required to read pictures from a digital camera, index the pictures by date, and store copies of them in an archive.
Data types:
  1. integer
  2. float
  3. string
  1. Numeric data consisting of whole numbers.
  2. Numeric data that might contain values other than whole numbers.
  3. Data representing non-numeric data that will generally not be manipulated by mathematical expressions. For instance Names, Zip Codes, Addresses, Phone Numbers.
Primitive data type Data types that are included as primitives in a programming language such as int for integer and char for character.
Data structure Variables in a program that determines the character or shape of the data. Ex: text is normally viewed as a long string of characters, and sales records might be viewed as a rectangular table of numeric values.
Python collection data structures:
  1. list
  2. tuple
  3. set
  4. dictionary
  1. Used to store multiple items in a single variable. Lists are created using [] square brackets.
  2. Used to store multiple items in a single variable. A tuple is a collection which is ordered and unchangeable. Tuples are written with () round brackets.
  3. Used to store multiple items in a single variable. A set is a collection with is unordered, unchangeable, and unindexed. Sets are written with {} curly brackets.
  4. Used to store data values in key:value pairs. A dictionary is a collection which is ordered, changeable and DO NOT allow duplicates. Dictionaries are written with {} curly brackets, and have keys and values.
Function A set of instructions for performing a task that can be used as an abstract tool by other program units.
Local variable A variable declared within a function.
Global variable A variable whose scope is not restricted to a particular part of a program.
Source program A program in its original form and language.
Object program translated version of a source program.
