Chapter 3: Operating Systems Terminology

Operating System:
The software that controls the overall operation of a computer. It provides the means by which a user can store and retrieve files, provides the interface by which a user can request the execution of programs, and provides the environment necessary to to execute the prorams requested.

Job Queue:
In batch processing systems, the jobs residing in mass storage wait for execution in a queue. This queue organizes jobs in a first-in, first-out fashion.

FIFO:
An abbreviation for "First-in, First-out" meaning objects are removed from the queue in the order in which they arrived.

Interactive Processing:
A feature of operating systems that allow a program that is being executed to carry on a dialogue with the user through remote terminals.

Time-sharing:
A feature in operating systems designed to provide service to multiple users at the same time.

Multiprogramming:
The implimentation of time-sharing, in which time is divided into intervals and then the execution of each job is restricted to only one interval at a time.

Aplication Software:
Consists of the programs for performing specific tasks particular to the machine's utilization.

System Software:
Performs tasks that are common to computer systems in general. The system software provides the infrastructure that the application software requires.

Utility Software:
Software consisting of programs for performing activities that are fundamental to the computer installations but not included in the operating system.

Graphical User Interface (GUI):
The means in which objects are to be manipulated, such as files and programs, and are represented pictorially on the display as icons.

Shell:
An older user interface that communicates with the user through textual messages using a keyboard and monitory screen.

Kernel:
The internal part of an operating system, containing software components that perform the very basic functions required by the computer installation.

Device Driver:
A software unit that communicates with the controllers to carry out operations on the peripheral device (such as a printer, disk,or monitor) attached to the machine.

File Manager:
Maintains records of all the files stored in mass storage, including where each file is located, which users are allowed to access the various files, and which portions of mass storage are available for new files or extensions to existing files.

Memory Manager:
A component of the kernel which is charged with the task of coordinating the machine's use of main memory.

Virtual Memory:
A common technique used in a computer's operating system. Virtual memory uses both hardware and software to enable a computer to compensate for physical memory shortages, temporarily transferring data from RAM to disk storage.

Boot Strapping:
This process is performed by a computer each time it is turned on. It is the procedure that transfers the operating system from mass storage (permanent storage location) into mail memory (which is empty when the machine is first turned on).

ROM:
Also known as "Read Only Memory". This is a small portion of the main memory where the CPU finds its intial program. This memory is constructed from special nonvolatile memory cells.

Deadlock:
The condition in which two or more processes are blocked from progressing because each is waiting for a resource that is allocated to another.

Spooling:
A technique for granting several processes access to a common resource. A file manager may allocate file access according to the needs of the processes, allowing several processes to have read access but allowing only one to have write access, to mitigate conflicts.

Super User (or Administrator):
This user gains highly privileged access to the operating system by identifying them as the admin during the login procedure. The administrator can alter settings within the operating system, modify critical software packages, adjust the privileges granted to other users, and perform a variety of other mainntenance activities that are denied normal users.

Multi-factor Authentication:
A security process at login, that requires the user to enter a password as well as a special access code from a specialized security token, a smartphone with a particular phone number, or some form of unique biometrics in order to prove the user is in fact who they say they are.
